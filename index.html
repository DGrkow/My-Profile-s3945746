Project Idea

Summary

For my project idea I’ve always been intrigued with Virtual Reality and ways that we can use it to assist people in their education experience. Thus, my project idea would be a Virtual Reality program that would allow users who are unable to attend on-campus to be able to view there lectures via VR. It would involve interactive activities, which would include quizzes, polls, and discussions as well as a live chat feature to allow direct communication with their peers and lecturer. I believe this would help people who are online be able to have a more hands on/practical experience, resulting in more engagement from the user, possibly assisting them in their studies. In addition, it will assist students who are more hands on learners to gain the learning experience they require. Overall, this project idea will assist in the education of users who are unable to attend on-campus lectures.

My Motivation

This project idea will be very useful in assisting students who can’t attend on-campus lectures. It will create a more interactive experience resulting in an increase of engagement levels from students. In addition, similar technology is being used already in the Stanford School of Business and the University of British Colombia Law School. Also, this technology is changing the way we deliver education, rather than sitting in a lecture it can be done all at home with the same high level education experience being provided. Thus, for students who are mostly learning only it provides them a way to have a collaborative experience rather than watching a recorded lecture that has little to no interactions.

Description

Within this product it will include many features including interactive quiz’s, pre-recorded lecture, a virtual environment (a lecture hall) and a live-chat with other students and the lecturer allowing for an interactive experience.

The interactive quiz will be implemented as a pop up display that the user will be able to view. Firstly, the pop up display needs to be prominent enough for the user to notice it and have all the relevant questions needed. To ensure that all the relevant questions are there the lecturer will be able to input questions before the lecture so that it allows the user to reflect on the information learnt throughout the lecture they are watching. It will require the user to input an answer most likely a multiple choice selection. The input will most likely be inputted through a keyboard (however using a keyboard may be difficult especially as the user won’t be able to see their keyboard) or motion controllers, by hovering over the choices and selecting one. The program will take the user’s input and determine whether the user’s input was correct or incorrect, then the program will output there results.

The pre-recorded lecture will have to be uploaded by the lecturer into the system beforehand. This will then be implemented into the product so that the user is able to watch it in the virtual environment, most likely on a virtual screen. To allow the lecturer to input a pre-recorded lecture into the virtual environment the product will have to include a graphical interface that allows to access files specified by the lecturer to be uploaded, this graphical interface would also have features to input power point slides and different learning materials so the students can have access to them during the lecture. Another feature would include an option to have the lecture live-stream directly into the virtual environment, this will allow the user to discuss relevant topics live. The live-streaming function would be directly linked with the live-chat feature where the lecturer and students would be able to cover and discuss different information and topics.

Live-chat with other students and the lecturer, this feature will be included within the product to allow the user to clarify different content that was covered within a lecture, or ask questions on a particular topic. This would be linked to a service such as collaborate ultra that will allow the user to input a message through the chat function or speak directly through there microphone. Other services that will assist in this feature would include in-built packages within unity that will allow for live video and chat within the product. Also, this live-chat feature could also be implemented into smaller group sessions such as working on assignments or presenting to a class, which will develop collaborative and communication skills.

A virtual environment will need to be created as a part of this product. It will allow the user to have an immersive experience and have an experience of being in a lecture hall. In addition, this virtual environment would include many different features as discussed earlier including the lecture and the quiz. Creating the virtual environment through software such as unity would require building and designing different objects to create the desired environment, in addition to objects it will require lighting and different textures to be implemented onto the different objects.

Tools and Technologies Required

The tools and technology that I require includes Unity, a VR headset and sensors. Unity is free software that is used to create and design games/digital environments such as a VR lecture. Unity has many add-ons to assist in making a VR program; this includes a live chat feature and specific types of objects. The VR headset would be required to use the VR program in addition to the headset motion controllers will be required to interact with the program which includes participating in the in-built quizzes, the sensors would be necessary to track the user’s motion in their own environment such as their bedroom.

If I was going to make a VR headset for this product, it would include different components such as the shell of the headset, using a 3D printer to build it. The shell would be used to house all the components; some of the components would include an Arduino Due, LCD screen, accelerometer, magnetometers and a gyroscope. The Arduino due allows for control of the computer system acts as a mini computer which controls the functions within the system. The LCD screen would be within the shell of the headset used to display the program. The accelerometer would be used for rotational tracking knowing which way is up, magnetometers is used to track/detect movement and speed and a gyroscope would have to be implemented to detect the tilt of the device.
